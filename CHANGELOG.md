# 更新日志

## 1.0.5 - 2026-07-10

### 重点更新

- 全面重构 TXT 转 EPUB 工作流，优化文件导入、章节识别、规则调整、预览检查和生成步骤。
- 所有功能页面统一为紧凑、简洁的工具界面，并为整个应用内置思源宋体。
- Windows 发布产物升级为可直接运行的 EXE 安装程序；Windows 与 macOS 均内置 WebP 转换组件。

### TXT 转 EPUB

- 新增阅微和 Kindle 两种章节头图，可自动插入每个章节标题前并生成对应 CSS。
- 新增阅微和 Kindle 两种全屏首页，首页图片可独立选择，不再与普通封面共用。
- 重构章节分割为顺序多规则流程，支持 1–6 级目录、最多 15 条正则、规则排序和独立分页控制。
- 导入 TXT 后自动扫描推荐章节规则，并显示每个标题匹配的规则、标题级别和原文行号。
- 支持整类关闭某条目录规则，也可以逐项取消误识别标题；不分页规则会保留为 H1–H6 页内标题。
- 章节规则卡片重新设计，使用清晰的 H1–H6 标识和“分页”状态标签，排序及删除操作统一收进规则菜单。
- 默认输出文件名改为 `书名-作者.epub`；作者为空时只使用书名，两者都为空时使用原 TXT 文件名。
- 修改书名或作者时自动同步默认输出路径，用户手动选择的保存位置不会被覆盖。
- 不再向生成的 EPUB 自动注入 5% 页面边距、指定正文字体和 1.8 行高。

### EPUB 工具

- 元数据编辑新增完整 OPF 源码查看、刷新和复制，原“查看 OPF”入口已合并移除。
- 字体加密在导入 EPUB 后自动扫描可加密的 TTF 字体族和正文文件，支持字体族筛选、全选及逐项选择。
- 原“列出字体目标”功能已合并到字体加密页面，减少重复操作。
- “阅微转多看”和“得到转多看”移动到“注释 / 注音”分类，移除空的“平台适配”分类。

### 界面与交互

- 内置思源宋体 CN Regular、SemiBold、Bold，所有页面、日志、正则和源码预览统一使用思源宋体。
- 所有功能页统一页头、描边卡片、文件选择行、输入框、下拉框和底部主操作按钮。
- 日志区改为“查看日志”按钮，弹窗支持滚动、文本选择、全选、复制全部和清空。
- 日志操作在窄窗口下自动收纳，所有功能页均完成窄窗口无溢出适配。
- 优化侧边栏、仪表盘和功能目录，功能名称、分组与数量保持一致。

### 修复

- 修复 Big5 文本容易被误判为 GBK、残缺字节可能中断导入以及 GB18030 四字节被错误解码的问题。
- 修复章节规则文字被分页开关遮挡、H1 标识和开关布局拥挤的问题。
- 修复字体扫描把不支持的 OTF、WOFF 字体误报为可加密目标的问题。
- 修复字体或正文未选择时仍可能执行加密的问题。
- 修复 Kindle 推送页卡片内开关点击反馈被背景遮挡的问题。

### 构建与发布

- 应用版本统一更新为 `1.0.5`，默认构建号为 `5`；侧边栏、各平台应用和安装程序版本保持一致。
- GitHub Actions 改为读取 `pubspec.yaml` 中的版本号，不再根据运行次数自动改变软件版本。
- Windows 自动发布改为单文件 EXE 安装程序，支持开始菜单、可选桌面快捷方式和卸载。
- Windows 安装程序内置 `cwebp 1.6.0`，无需额外安装组件或配置环境变量。
- macOS 应用内置 `cwebp` 及所需动态库，并附带 libwebp 许可证和完整签名校验。
- 修复 Inno Setup 缺少简体中文语言文件导致 Windows 自动打包失败的问题。

### 测试

- 新增 TXT 输出命名、多级分章、规则优先级、误识别排除、页内标题和无效正则回归测试。
- 新增阅微/Kindle 头图、全屏首页、资源清单、阅读顺序、CSS 和图片尺寸测试。
- 新增字体扫描、字体加密、内置字体、日志弹窗以及全部功能页窄窗口测试。
- 使用真实 EPUB 验证字体加密和主要处理流程；当前完整测试为 105 项通过，17 项因缺少可选本机夹具按设计跳过。

## 1.0.2 - 2026-07-10

### 新增

- TXT 转 EPUB 默认输出名改为“书名-作者.epub”；作者为空时只使用书名，书名和作者都为空时回退到原 TXT 文件名，并自动处理跨平台非法文件名字符。
- TXT 编码读取新增完整 GBK 与 Big5 解码支持。
- TXT 转 EPUB 新增可选章节头图，可选择 PNG/JPG 图片并使用阅微通栏或 Kindle 越界页边距样式自动插入每个章节标题前。
- TXT 转 EPUB 新增全屏首页，可独立选择阅微 1080×2400 或 Kindle 1536×2048 首页图片，并自动置于阅读顺序第一页，不与普通封面互相覆盖。
- TXT 分章重构为顺序多规则流程：导入后自动组合匹配规则，支持 1–6 级目录、最多 15 条可编辑正则、单规则分页开关及规则优先级调整。
- TXT 分章预览新增标题匹配清单，可直接取消误识别项；不分页规则会在章节内生成对应 H1–H6 标题。
- TXT 分章预览支持按正则类型整类关闭目录识别，关闭后该类标题统一恢复为普通正文。
- TXT 分章的每个预览标题都会显示匹配的规则名称、完整正则、标题级别和源文本行号。
- TXT 转 EPUB 的所有输出模式不再自动注入全局 `body` 的 5% 边距、指定字体和 1.8 行高。
- 重整 TXT 转 EPUB 设置页：文件信息与图片设置使用分组卡片，全屏首页和头图选项按需展开，并支持窄窗口自动换行。
- 字体加密页面在导入 EPUB 后自动扫描可加密的 TTF 字体族和正文文件。
- 字体扫描结果支持字体族筛选、正文文件逐项选择、全选和清空。
- 元数据编辑页面新增完整 OPF 源码查看、刷新和复制功能。
- Windows 自动发布改为单文件安装程序 `epub-toolkit-windows-*-setup.exe`，支持开始菜单、可选桌面快捷方式和卸载。

### 修复

- 修复 TXT 转 EPUB 修改书名或作者后默认输出路径不更新的问题；用户手动选择保存位置后不再被自动命名覆盖。
- 修复 Big5 字节范围与 GBK 重叠时总被识别为 GBK、导致繁体 TXT 乱码的问题。
- 修复 Big5 遇到残缺字节时可能中断导入的问题，现在使用替换字符容错。
- 停止将 GB18030 四字节序列错误地交给 GBK 解码器，避免静默生成乱码。
- 字体扫描仅列出当前字体加密器实际支持的 TTF 字体，不再把 OTF、WOFF 误报为可加密目标。
- 字体或正文选择为空时阻止执行，避免空选择被误解释为加密全部内容。

### 界面调整

- 应用内置思源宋体 CN 的 Regular、SemiBold、Bold 字重，所有页面、日志、正则和源码预览统一使用思源宋体，不再依赖系统字体。
- 重构 TXT 章节分割规则卡片：优化 H1–H6 层级标识，以“分页”状态标签替代拥挤的开关，将排序和删除收纳到规则菜单，并让正则输入区独占整行避免文字遮挡。
- 所有功能页统一采用 TXT 转 EPUB 的紧凑布局：缩小页头和控件占用，统一扁平描边卡片、文件选择行、输入框与底部主操作栏，并改善窄窗口适配。
- 页面内日志区改为单一“查看日志”按钮；点击后通过弹窗查看日志，支持滚动、文本选择、全选、复制全部和清空，窄窗口下自动收纳操作项。
- 将独立的“查看 OPF”功能合并到“元数据编辑”，移除原入口、页面和路由。
- 将独立的“列出字体目标”功能合并到“字体加密”，移除原入口、页面和路由。
- 将“阅微转多看”和“得到转多看”移动到侧边栏“注释 / 注音”分类，并移除空的“平台适配”分类。
- 同步更新仪表盘和工具目录中的功能名称、说明与数量。

### 构建与发布

- Windows 安装程序内置官方 `cwebp 1.6.0`，安装后可离线使用图片转 WebP，无需另装组件或配置环境变量。
- Windows 构建会校验 libwebp 下载文件的 SHA-256，并在打包前运行 `cwebp.exe -version` 做可执行验证。
- macOS GitHub Actions 自动安装并内置 `cwebp` 及所需动态库，移除对 Homebrew 路径的运行时依赖。
- macOS 应用内附带 libwebp 许可证，加入组件后重新签名并执行深度签名验证。
- 发布说明中的 Windows 产物由 ZIP 更新为安装程序 EXE；未签名安装程序首次运行时可能显示 SmartScreen 提示。
- 修复 GitHub Actions 中 Inno Setup 缺少简体中文语言文件导致 Windows 安装程序编译失败的问题；构建时从官方固定版本获取并校验文件哈希。

### 测试

- 新增 TXT 转 EPUB 书名、作者、原文件名回退和非法字符处理的输出命名测试。
- 新增内置思源宋体资源、主题字体及章节规则卡片控件回归测试。
- 新增日志弹窗完整交互测试，并逐页验证全部功能页在窄窗口下可正常构建、无布局溢出。
- 新增 GBK、Big5 自动检测与完整解码、GB18030 防误解码及 Big5 残缺字节回归测试。
- 新增 TXT 转 EPUB 阅微/Kindle 头图资源、全屏首页、OPF 清单与 spine 首项、章节 XHTML、CSS、尺寸校验及未启用兼容性测试。
- 新增多级分章、规则优先级、页内标题、误识别排除、无效正则及空行位置回归测试。
- 扩充字体目标结构化扫描测试，并覆盖扫描结果与字体加密参数的衔接。
- 使用约 30 MB 的真实 EPUB 验证字体加密：识别 1 个字体族和 34 个正文文件，成功修改 1 个 TTF，并生成 359 个正文混淆字符。
- macOS Release 构建、内置 `cwebp`、PNG 转 WebP、动态库引用及 App 深度签名均已通过本机验证。
- 完整 Flutter 测试通过；依赖本机真实书籍的测试在夹具缺失时会明确跳过。

## 1.0.0 - 2026-07-09

### 新增

- 将项目初始发布版本设置为 `1.0.0+1`。
- 新增 GitHub Actions 自动构建流程。
- 支持在 GitHub 上自动构建 Android APK、macOS 应用、Windows 应用和 iOS 无签名应用包。
- 自动构建完成后创建 GitHub Release，并上传各平台构建产物。
- 构建版本号从 `1.0.0` 开始，后续每次 GitHub Actions 构建按 `1.0.1`、`1.0.2` 递增。

### 调整

- 调整 Android 忽略规则，允许提交 CI 构建必需的 Gradle Wrapper 文件。
- 保持签名文件和本地配置文件被忽略，避免上传 keystore、`local.properties` 等敏感或本机专用文件。
- macOS 本机复制脚本在 GitHub Actions 环境中自动跳过，避免 CI 构建因访问本机路径失败。
- 使用图标源文件更新 Android、iOS、macOS、Windows 和 Web 应用图标。
- 根据 README 的功能分组重新设计仪表盘 UI。

### 清理

- 新增根目录 `.gitignore`，忽略 Flutter/Dart 构建产物、本地缓存和发布包目录。
- 清理本地编译产物、发布包、平台缓存和不可达 Git 大对象。
- 删除已不再需要的 `icons/` 图标源文件夹。

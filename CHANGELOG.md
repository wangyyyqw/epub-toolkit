# 更新日志

## 1.0.2 - 2026-07-10

### 新增

- TXT 编码读取新增完整 GBK 与 Big5 解码支持。
- 字体加密页面在导入 EPUB 后自动扫描可加密的 TTF 字体族和正文文件。
- 字体扫描结果支持字体族筛选、正文文件逐项选择、全选和清空。
- 元数据编辑页面新增完整 OPF 源码查看、刷新和复制功能。
- Windows 自动发布改为单文件安装程序 `epub-toolkit-windows-*-setup.exe`，支持开始菜单、可选桌面快捷方式和卸载。

### 修复

- 修复 Big5 字节范围与 GBK 重叠时总被识别为 GBK、导致繁体 TXT 乱码的问题。
- 修复 Big5 遇到残缺字节时可能中断导入的问题，现在使用替换字符容错。
- 停止将 GB18030 四字节序列错误地交给 GBK 解码器，避免静默生成乱码。
- 字体扫描仅列出当前字体加密器实际支持的 TTF 字体，不再把 OTF、WOFF 误报为可加密目标。
- 字体或正文选择为空时阻止执行，避免空选择被误解释为加密全部内容。

### 界面调整

- 将独立的“查看 OPF”功能合并到“元数据编辑”，移除原入口、页面和路由。
- 将独立的“列出字体目标”功能合并到“字体加密”，移除原入口、页面和路由。
- 将“阅微转多看”和“得到转多看”移动到侧边栏“注释 / 注音”分类，并移除空的“平台适配”分类。
- 同步更新仪表盘和工具目录中的功能名称、说明与数量。

### 构建与发布

- Windows 安装程序内置官方 `cwebp 1.6.0`，安装后可离线使用图片转 WebP，无需另装组件或配置环境变量。
- Windows 构建会校验 libwebp 下载文件的 SHA-256，并在打包前运行 `cwebp.exe -version` 做可执行验证。
- macOS GitHub Actions 自动安装并内置 `cwebp` 及所需动态库，移除对 Homebrew 路径的运行时依赖。
- macOS 应用内附带 libwebp 许可证，加入组件后重新签名并执行深度签名验证。
- 发布说明中的 Windows 产物由 ZIP 更新为安装程序 EXE；未签名安装程序首次运行时可能显示 SmartScreen 提示。

### 测试

- 新增 GBK、Big5 自动检测与完整解码、GB18030 防误解码及 Big5 残缺字节回归测试。
- 扩充字体目标结构化扫描测试，并覆盖扫描结果与字体加密参数的衔接。
- 使用约 30 MB 的真实 EPUB 验证字体加密：识别 1 个字体族和 34 个正文文件，成功修改 1 个 TTF，并生成 359 个正文混淆字符。
- macOS Release 构建、内置 `cwebp`、PNG 转 WebP、动态库引用及 App 深度签名均已通过本机验证。
- 完整 Flutter 测试通过；依赖本机真实书籍的测试在夹具缺失时会明确跳过。

## 1.0.0 - 2026-07-09

### 新增

- 将项目初始发布版本设置为 `1.0.0+1`。
- 新增 GitHub Actions 自动构建流程。
- 支持在 GitHub 上自动构建 Android APK、macOS 应用、Windows 应用和 iOS 无签名应用包。
- 自动构建完成后创建 GitHub Release，并上传各平台构建产物。
- 构建版本号从 `1.0.0` 开始，后续每次 GitHub Actions 构建按 `1.0.1`、`1.0.2` 递增。

### 调整

- 调整 Android 忽略规则，允许提交 CI 构建必需的 Gradle Wrapper 文件。
- 保持签名文件和本地配置文件被忽略，避免上传 keystore、`local.properties` 等敏感或本机专用文件。
- macOS 本机复制脚本在 GitHub Actions 环境中自动跳过，避免 CI 构建因访问本机路径失败。
- 使用图标源文件更新 Android、iOS、macOS、Windows 和 Web 应用图标。
- 根据 README 的功能分组重新设计仪表盘 UI。

### 清理

- 新增根目录 `.gitignore`，忽略 Flutter/Dart 构建产物、本地缓存和发布包目录。
- 清理本地编译产物、发布包、平台缓存和不可达 Git 大对象。
- 删除已不再需要的 `icons/` 图标源文件夹。
